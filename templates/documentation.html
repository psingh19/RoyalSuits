Motive

My motive behind this website is to implement most of the stuff that I learnt in CPSC 2030. This includes AJAX functionality, php, jQuery and aria stuff. It is a dynamic website made from scratch.

Features

  • The blog page contains a functionality like a facebook feed that is refreshed in every 10s.
  • I have used a FadeIn keyframe on main element.
  • The shop button do have shadow transition effect in Desktop view. In mobile view, the left side icon pop-up a menu with transition
  • The site is responsive with any kind of device. The header closes all nav items into a button.
  • I have used lot of grids. For example - The social icons are placed in a grid, the collection page elements are placed in a grid.

Requirements

  • Organization: The architecture of this site is tree-based. Firstly, index.php (i.e. home-page) is called which then calls the single.php for all other pages. All my HTML code is placed inside "templates" folder. The database connection stuff is inside the "database" folder. The form structure on the "Blog" page uses post-form.css for style.

  • Dynamic content: The "Blog" page has dynamic content in it i.e. connection with a database. The "form" in this page stores data into the database with AJAX post request and the data is fetched using AJAX get request and displayed in the comment section.

  • One form: All the "blog" posts have a form with them and the form does have client-side as well as server-side validation in order to prevent malicious activity or wrong input.

  • Update the database: User can easily update database on the "blog" page. This page has a review section on each post that enables them to update database anytime.

  • Security Measure: As I said this website has a client side as well as a server side validation on form that prevents users to enter wrong input. Besides this, I used prepare statements while updating the database that prevent SQL injection.
